# Leave Manager — 스타트업 친화적 조직 연차 관리 시스템

## 1. 프로젝트 개요

소규모~중견 조직(20~200명)이 엑셀·구글시트·카카오톡으로 처리하던 연차 신청·기록·승인 흐름을, 웹 한 화면으로 정리하기 위해 만든 자체 사내 SaaS다. Next.js 14 App Router + Supabase Postgres + NextAuth(Google OAuth) 위에 얹었고, Slack/Email 알림과 Google Calendar 연동이 기본으로 들어간다. 별도 라이선스 비용이 큰 HRIS를 도입할 여유가 없는 스타트업을 타깃으로 한다.

## 2. 만든 이유 (Problem)

- 엑셀 시트로 연차를 관리하다 보니 **이중 입력·집계 오류·승인 라인 누락**이 반복됐다.
- 연차 잔여일수를 직원이 직접 묻고 인사 담당자가 매번 계산해 회신하는 구조 → 인사팀 시간 소비.
- "이번 달 누가 쉬는가"를 한눈에 보기 어려워 회의·일정 충돌이 잦았다.
- 상용 HR 솔루션(Flex·시프티·잡플래닛 등)은 30인 미만 기준 월 5~30만원으로 부담스럽고, 우리 회사 룰(반차 단위·이월 정책·회계연도 기준)에 정확히 안 맞는다.

## 3. 솔루션

| 영역 | 결정 사항 |
|---|---|
| 인증 | NextAuth + Google OAuth — 회사 도메인 한정 로그인 |
| DB | Supabase Postgres — 무료 티어 500MB 충분, RLS는 비활성화하고 서버 사이드 API 라우트로 권한 통제 |
| 권한 | `employees.role` 단일 컬럼 (`admin` / `user`) — 1~2명 인사 담당자만 admin |
| 단위 | `full` / `am-half` / `pm-half` — 반차 0.5일 단위 지원 |
| 보정 | `leave_adjustments` 별도 테이블 — 이월/추가부여/차감 이력 모두 보존 |
| 알림 | Resend(이메일) + Slack Web API — 신청·승인 시 자동 발송 |
| 일정 공유 | Google Calendar API — 승인된 연차를 회사 캘린더에 자동 등록 |

## 4. 핵심 기능

### 4.1 일반 사용자 (`/my`)
- 본인 연차 잔여일·사용일·이월일 한눈에 표시
- 캘린더 클릭으로 연차/오전반차/오후반차 신청
- 신청 즉시 인사·관리자에게 Slack DM + 이메일 발송
- 본인 연차 사용 이력 타임라인

### 4.2 관리자 (`/admin`)
- 전체 직원 목록·근속·연차 잔여 일괄 조회
- 월별 연차 사용 현황 캘린더 (이번 달 누가 쉬는가)
- 직원별 연차 보정 (이월·추가부여·차감) — 사유 필수
- CSV 내보내기 (연말 정산용)
- 신규 입사자 등록 (이메일·이름·입사일·역할)

### 4.3 알림·연동
- **Slack**: 연차 신청 시 #hr 채널 + 매니저 DM
- **이메일**: 신청자/관리자에게 Resend 기반 트랜잭셔널 메일
- **Google Calendar**: 승인 즉시 회사 공용 캘린더에 "○○님 연차" 일정 등록

## 5. 데이터 모델

```
employees
  ├ id (UUID)
  ├ email (UNIQUE)        ← 회사 도메인 검증
  ├ name
  ├ hire_date              ← 연차 발생 기준일
  ├ role (admin/user)
  └ is_active

leaves
  ├ employee_id → employees
  ├ date
  ├ type (full/am-half/pm-half)
  ├ leave_year             ← 회계연도 단위 집계
  └ created_by → employees ← 관리자가 대신 등록한 경우 추적

leave_adjustments
  ├ employee_id → employees
  ├ leave_year
  ├ adjustment (NUMERIC 3,1)  ← 음수 가능 (차감)
  ├ reason
  └ created_by
```

`UNIQUE(employee_id, date, type)` 제약으로 중복 등록 차단.

## 6. 기술 스택

- 프론트: Next.js 14 App Router + TypeScript + TailwindCSS
- 인증: NextAuth.js (Google OAuth Provider)
- DB·BaaS: Supabase (Postgres)
- 알림: Resend, @slack/web-api
- 캘린더: googleapis (Calendar API)
- 배포: Vercel (서버리스 함수 + Edge)

## 7. 운영 결과 / 효과

- **인사팀 처리 시간 절감**: 연차 잔여 문의·집계 응답 → 0건/주.
- **승인 누락 0**: Slack/이메일 자동 발송으로 결재 누락 사고 차단.
- **회의 충돌 감소**: "이번 주 출근자" 캘린더 뷰로 일정 협의 시간 단축.
- **운영비**: Supabase 무료 + Vercel 무료 + Resend 100건/일 무료 티어 안에서 200명 규모까지 커버 가능.

## 8. 향후 개선 방향

- **결재 라인 다단계화** — 현재는 admin 단일 승인. 사업부장 → 인사 2단 결재 옵션 추가
- **휴가 종류 확장** — 경조사·병가·재택 등 휴가 유형 분리
- **연차 자동 부여 스케줄러** — Vercel Cron으로 매년 1월/입사 기준일에 자동 부여
- **재직증명서 자동 발급** — `employees` 데이터를 PDF로 출력
- **연동 확장** — Notion/Jira 캘린더 동기화

## 9. 한계 / 알려진 이슈

- 회사 도메인 1개 기준 설계 — 다도메인 그룹사는 추가 설계 필요.
- Supabase RLS 비활성화 + 서버 API에서 권한 검증 — 환경변수 유출 시 위험. 서비스 키는 절대 클라이언트 노출 금지.
- 시간 단위 연차(1시간·2시간)는 미지원.
- 회계연도(`leave_year`)는 양의 정수 1개만 가짐. 회계연도 변경 시 마이그레이션 스크립트 필요.
